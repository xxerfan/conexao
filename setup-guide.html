Pré-requisitos & Anon Key

Verifique sua conta Supabase e atualize a chave de API no sistema

Erro HTTP 401? Isso significa que a chave de API (anon key) está incorreta ou expirada. Siga os passos abaixo.
Passo a passo
  1. 1
    Acesse seu projeto no Supabase Vá em supabase.com/dashboard e clique no seu projeto
  2. 2
    Encontre a Anon Key No menu lateral clique em Project Settings → API. Procure a linha "anon public" e copie o valor inteiro (começa com eyJ...)
  3. 3
    Encontre a Project URL Na mesma tela, copie a Project URL — formato: https://xxxxxxxxxxxx.supabase.co
  4. 4
    Atualize o arquivo js/api.js Abra o arquivo e localize as 2 constantes abaixo — cole os valores copiados
Constantes para atualizar em js/api.js
// ── Linha 1-2 do arquivo js/api.js ──────────────────────
const SUPABASE_URL = 'https://SEU_PROJETO.supabase.co';
const SUPABASE_KEY = 'eyJhbGc...SUA_ANON_KEY_COMPLETA...';
A anon key começa com eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9 — se começar com sb_publishable_ é o formato novo; verifique se o seu projeto está usando o SDK v2 ou rode o diagnóstico abaixo.
Teste rápido de conexão
Clique em "Testar" para verificar a conexão

Executar SQL — Criação das Tabelas

Cria todas as tabelas, RLS, triggers e sistema de convites

Este passo é o mais importante. Execute o SQL completo uma única vez. O script é idempotente (pode ser re-executado sem duplicar dados).
Como executar o SQL
  1. 1
    Abra o SQL Editor no Supabase Menu lateral → SQL Editor → clique em "+ New query"
  2. 2
    Copie o conteúdo de supabase_setup.sql Abra o arquivo no editor de código, selecione tudo (Ctrl+A) e copie (Ctrl+C)
  3. 3
    Cole no SQL Editor e clique em RUN Cole o código (Ctrl+V) na área de query e clique no botão verde "RUN" (ou F5)
  4. 4
    Verifique os resultados O script retornará uma tabela mostrando todas as tabelas com ✓ user_id e as políticas RLS ativas
Tabelas que serão criadas
ong_projetos
Projetos/convênios
ong_rubricas
Rubricas orçamentárias
ong_despesas
Lançamentos financeiros
ong_metas
Metas físicas
ong_cronograma
Cronograma de atividades
ong_documentos
Documentos e anexos
ong_auditoria
Log de auditoria MROSC
ong_convites
Códigos de autorização
O que o SQL configura automaticamente
RLS (Row Level Security) — cada ONG vê apenas seus próprios dados via auth.uid() = user_id
Trigger automático — valor_executado nas rubricas é recalculado automaticamente ao lançar despesas
Auditoria — todas as mudanças em despesas, rubricas e projetos são registradas na ong_auditoria
3 códigos iniciais de convite já criados: ONG2024, ADMIN001, MROSC2024

Configurar Autenticação

Ative o provedor de e-mail e configure os URLs corretos

Passo 1 — Ativar Email Provider
  1. 1
    Acesse Authentication → Providers Supabase Dashboard › Authentication › Providers
  2. 2
    Clique em "Email" e certifique que está habilitado O toggle deve estar verde (ON). Habilite também a opção "Enable email confirmations" — recomendado para segurança
Passo 2 — Configurar Site URL
  1. 1
    Acesse Authentication → URL Configuration Authentication › URL Configuration
  2. 2
    Site URL Coloque a URL onde o sistema está hospedado. Em desenvolvimento use: http://localhost:3000
  3. 3
    Redirect URLs Adicione a URL do seu site + /index.html. Em produção será algo como https://seusite.genspark.site/index.html
Passo 3 — Template de e-mail em português
Opcional mas recomendado: personalize o e-mail de confirmação em português para seus usuários.
  1. 1
    Acesse Authentication → Email Templates Authentication › Email Templates
  2. 2
    Edite o template "Confirm signup" Substitua o conteúdo pelo HTML abaixo

Criar Bucket de Storage

Para armazenar notas fiscais, documentos e logos das ONGs

O bucket precisa ser criado manualmente pelo dashboard do Supabase — não é possível criar via SQL. O SQL já criou as políticas de acesso.
Como criar o bucket
  1. 1
    Acesse Storage no Supabase Supabase Dashboard › Storage
  2. 2
    Clique em "New bucket" Botão no canto superior direito ou "+ New bucket"
  3. 3
    Nome do bucket: ong-arquivos ⚠ Digite exatamente esse nome (com hífen, sem espaços, tudo minúsculo)
  4. 4
    Configurações do bucket Deixe Private (não público) — as políticas RLS já controlam o acesso por usuário
  5. 5
    Tamanho máximo de arquivo Recomendado: 10 MB (suficiente para PDFs e imagens de NF)
  6. 6
    Clique em "Save" ou "Create bucket" O bucket aparecerá na lista com o ícone de pasta
Estrutura de pastas no bucket

As pastas são criadas automaticamente ao fazer o primeiro upload:

ong-arquivos/
├── {user_id}/
│   ├── nf/          ← Notas Fiscais (PDF, JPG, PNG)
│   ├── logos/       ← Logos das ONGs (PNG, JPG)
│   └── documentos/  ← Documentos e anexos (PDF, DOC)

Controle quem pode criar conta no sistema usando códigos de convite

Como funciona: Para criar conta no ONG Gestor, o usuário precisa de um código de convite válido. Você gera os códigos aqui e distribui para as ONGs que terão acesso. Cada código é de uso único — após o primeiro cadastro, é desativado automaticamente.

Gerenciar convites diretamente via SQL

Execute os comandos abaixo no SQL Editor do Supabase para gerenciar convites sem precisar de autenticação:

▶ Criar convite simples:

INSERT INTO public.ong_convites (codigo, ativo, descricao, expira_em)
VALUES ('MEU_CODIGO', true, 'Descrição do convite', NOW() + INTERVAL '30 days');

▶ Criar convite para e-mail específico:

INSERT INTO public.ong_convites (codigo, ativo, email_destino, descricao, expira_em)
VALUES ('ONG_EXCLUSIVO', true, 'contato@ong.org.br', 'Convite exclusivo', NOW() + INTERVAL '7 days');

▶ Listar todos os convites:

SELECT codigo, ativo, email_destino, expira_em, usado_por, usado_em, descricao
FROM public.ong_convites
ORDER BY criado_em DESC;

▶ Revogar/desativar um convite:

UPDATE public.ong_convites SET ativo = false WHERE codigo = 'MEU_CODIGO';

▶ Gerar 5 convites aleatórios de uma vez:

INSERT INTO public.ong_convites (codigo, ativo, descricao, expira_em)
SELECT
  'ONG' || UPPER(SUBSTRING(gen_random_uuid()::text, 1, 8)),
  true,
  'Convite em lote #' || gs,
  NOW() + INTERVAL '90 days'
FROM generate_series(1, 5) AS gs;

Primeiro Login e Cadastro

Como criar a primeira conta e acessar o sistema

Fluxo de cadastro
  1. 1
    Acesse o ONG Gestor (index.html) A tela de login aparecerá automaticamente
  2. 2
    Clique em "Criar conta gratuita" O painel de cadastro será exibido
  3. 3
    Preencha: Nome da ONG, E-mail, Senha A senha deve ter no mínimo 6 caracteres
  4. 4
    Insira o Código de Convite Use um dos códigos criados: ONG2024 ou ADMIN001
  5. 5
    Clique em "Criar Conta" Se a confirmação de e-mail estiver ativada, verifique sua caixa de entrada e clique no link de confirmação
  6. 6
    Faça login Após confirmar o e-mail, volte ao sistema e faça login com e-mail e senha
Dica: Desativar confirmação de e-mail (dev)
Em ambiente de desenvolvimento, você pode desativar a confirmação de e-mail para agilizar testes. Não faça isso em produção.
  1. 1
    Acesse Authentication → Providers → Email
  2. 2
    Desative "Enable email confirmations" O usuário será logado imediatamente após o cadastro
Recuperar senha

Na tela de login, clique em "Esqueci minha senha". Um e-mail com link de redefinição será enviado automaticamente via Supabase Auth.

Diagnóstico de Conexão

Verifica todos os componentes do sistema automaticamente

Diagnóstico completo
Soluções para erros comuns
HTTP 401 Unauthorized

Anon key incorreta ou expirada. Vá em Project Settings → API e copie a nova anon public key. Atualize em js/api.js.

HTTP 404 Not Found

Tabela não existe. Execute o SQL completo (supabase_setup.sql) no SQL Editor do Supabase.

HTTP 406 Not Acceptable

Header Accept incorreto. Certifique que o código está enviando Accept: application/json.

HTTP 403 Forbidden (RLS)

RLS bloqueando acesso. Certifique que o usuário está logado e que o user_id está preenchido nos registros. Execute a migração de dados se necessário.

CORS Error Cross-Origin Request Blocked

URL do projeto Supabase incorreta em js/api.js. Verifique a SUPABASE_URL — não pode ter barra no final.

Migrar Dados Existentes

Se você já tinha dados sem user_id, vincule-os ao seu usuário

Só execute se necessário: se o banco estava vazio antes da configuração, pule esta etapa. Esta migração é para quem já tinha dados inseridos antes de ativar o RLS.
Buscar User ID

Primeiro, obtenha o UUID do seu usuário no Supabase: 

-- Execute no SQL Editor para ver seus usuários
SELECT id, email, created_at
FROM auth.users
ORDER BY created_at DESC
LIMIT 10;
Script de Migração

Substitua SEU_USER_ID_AQUI pelo UUID obtido acima e execute no SQL Editor: 

DO $$
DECLARE
  v_uid UUID := 'SEU_USER_ID_AQUI';
BEGIN
  -- Migra todos os registros sem user_id para o seu usuário
  UPDATE public.ong_projetos        SET user_id = v_uid WHERE user_id IS NULL;
  UPDATE public.ong_rubricas        SET user_id = v_uid WHERE user_id IS NULL;
  UPDATE public.ong_despesas        SET user_id = v_uid WHERE user_id IS NULL;
  UPDATE public.ong_metas           SET user_id = v_uid WHERE user_id IS NULL;
  UPDATE public.ong_cronograma      SET user_id = v_uid WHERE user_id IS NULL;
  UPDATE public.ong_documentos      SET user_id = v_uid WHERE user_id IS NULL;
  UPDATE public.ong_fases           SET user_id = v_uid WHERE user_id IS NULL;
  UPDATE public.ong_plano_aplicacao SET user_id = v_uid WHERE user_id IS NULL;

  RAISE NOTICE '✅ Migração concluída para user_id: %', v_uid;
END $$;
Verificação pós-migração
-- Verifica se ainda há registros sem user_id
SELECT
  'ong_projetos' AS tabela,
  COUNT(*) FILTER (WHERE user_id IS NULL) AS sem_uid,
  COUNT(*) AS total
FROM public.ong_projetos
UNION ALL SELECT 'ong_rubricas', COUNT(*) FILTER (WHERE user_id IS NULL), COUNT(*) FROM public.ong_rubricas
UNION ALL SELECT 'ong_despesas', COUNT(*) FILTER (WHERE user_id IS NULL), COUNT(*) FROM public.ong_despesas
UNION ALL SELECT 'ong_metas',    COUNT(*) FILTER (WHERE user_id IS NULL), COUNT(*) FROM public.ong_metas;
-- sem_uid deve ser 0 em todas as linhas
Configuração concluída!
